# 📚 Research Paper Assistant

A Retrieval-Augmented Generation (RAG) app that lets you upload research papers and ask questions about them in natural language. Answers are generated by Google's Gemini model and are always grounded in your documents, with every response traceable back to the exact PDF, page, and paragraph it came from.

**🔗 Live demo:** [research-assistant-sm.streamlit.app](https://research-assistant-sm.streamlit.app/)

## Features

- **Multi-PDF support** — upload one or several research papers in a single session and query across all of them at once
- **Source-grounded answers** — the model is instructed to answer only from retrieved context, and explicitly says so when an answer isn't found in the uploaded documents
- **Full citation trail** — every answer comes with an expandable panel showing the source PDF name, page number, and the exact retrieved paragraph
- **Conversational interface** — built on Streamlit's chat UI, with persistent chat history for the session
- **Privacy-conscious processing** — uploaded PDFs are read into memory and discarded immediately after text extraction; nothing is written to persistent disk storage

## Tech Stack

| Layer | Technology |
|---|---|
| UI | [Streamlit](https://streamlit.io/) |
| Orchestration | [LangChain](https://www.langchain.com/) |
| LLM | Google Gemini (`gemini-2.5-flash`) |
| Embeddings | Google Gemini Embeddings (`gemini-embedding-001`) |
| Vector store | [FAISS](https://github.com/facebookresearch/faiss) |
| PDF parsing | [pypdf](https://pypi.org/project/pypdf/) |

## How It Works

1. Uploaded PDFs are parsed page-by-page and tagged with their source filename.
2. Text is split into overlapping chunks (1000 characters, 200 overlap) to preserve context across boundaries.
3. Each chunk is embedded with Gemini's embedding model and indexed in an in-memory FAISS vector store.
4. On each question, the top 3 most relevant chunks are retrieved and passed to Gemini as context, alongside a strict instruction to answer only from that context.
5. The answer is returned along with the source chunks, which are displayed to the user for verification.

## Limitations

- Uploaded documents and chat history are session-scoped — refreshing the page or restarting the app clears the index, so it isn't a persistent knowledge base across visits.
- Answer quality depends on retrieval; very long or poorly formatted PDFs may produce weaker chunk boundaries.
- No authentication or rate limiting is currently in place; treat the public link accordingly.
